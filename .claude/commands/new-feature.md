# 新機能実装

以下の手順で新機能を実装してください。

1. 要件を整理して実装計画を提示する（実装前に必ず確認を取る）
2. 既存のファイル構成・DBスキーマと整合性を確認する
3. TypeScriptで型定義を先に作成する
4. コンポーネント・ロジックを実装する
5. Supabaseのスキーマ変更が必要な場合はSQLを提示する
6. 実装完了後に `git add . && git commit` まで行う

## 制約
- MVPを超える機能は追加しない
- 既存のデザインシステム（navy/orange/cream）に合わせる
- `.env.local` は触らない
- 不明点は勝手に補完せず確認する
